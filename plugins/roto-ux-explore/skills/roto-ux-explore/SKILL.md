# UX Explorer

페르소나의 관점에서 실제 브라우저로 사이트를 탐색하며 UX 개선점을 발견하는 스킬입니다.

Claude Code가 직접 스크린샷을 보고 판단하므로 별도 API 키가 필요 없습니다.

**모든 출력은 한국어로 작성한다.**

## 사전 요구사항

- `playwright`가 프로젝트 또는 글로벌에 설치되어 있어야 합니다.
- `tsx`가 프로젝트 또는 글로벌에 설치되어 있어야 합니다.
- 없으면 이 플러그인의 `scripts/` 디렉토리에서 `npm install`을 실행하세요.

## 사용 방법

```text
/roto-ux-explore <페르소나> <시작URL> [목표] [--repeat=N]
```

| 인자 | 설명 | 기본값 |
|------|------|--------|
| `페르소나` | 탐색할 사용자 유형 (필수) | - |
| `시작URL` | 탐색 시작 페이지 (필수) | - |
| `목표` | 달성할 목표 (선택) | 페르소나 기본 목표 사용 |
| `--repeat=N` | 반복 실행 횟수 (선택) | 1 (단일 실행) |

### 예시

```text
/roto-ux-explore 관객 http://localhost:3000
/roto-ux-explore 관객 https://example.com
/roto-ux-explore 관객 https://example.com --repeat=5
/roto-ux-explore 신규사용자 http://localhost:3000/signup "회원가입 완료하기"
/roto-ux-explore "처음 방문한 40대 주부, 온라인 쇼핑 경험 적음" https://shop.example.com
```

localhost와 프로덕션 URL 모두 지원한다.

## 페르소나 정의

### 기본 페르소나 3종

#### 신규사용자 (new-user)
- **배경**: 서비스를 처음 방문한 사람.
- **기본 목표**: "이 서비스가 뭘 하는 곳인지 파악하고, 핵심 기능을 찾고 싶다"
- **행동 패턴**: 홈 -> 둘러보기 -> 핵심 기능 탐색
- **주의점**: 전문 용어를 모름. 직관적인 UI를 기대함.

#### 재방문자 (returning-user)
- **배경**: 서비스를 몇 번 사용해본 사람.
- **기본 목표**: "이전에 사용했던 기능을 다시 찾아서 사용하고 싶다"
- **행동 패턴**: 로그인 -> 자주 쓰는 기능 접근
- **주의점**: 반복 작업의 효율성 중요.

#### 외국인 (foreigner)
- **배경**: 한국어를 모르는 외국인.
- **기본 목표**: "이 서비스를 영어(또는 다른 언어)로 이용하고 싶다"
- **행동 패턴**: 언어 전환 시도 -> 기능 탐색
- **주의점**: 번역 품질에 민감. 언어 전환 접근성.

### 커스텀 페르소나
위 3개 외에 자유 텍스트로 페르소나를 정의할 수 있다:
```text
/roto-ux-explore "처음 방문한 40대 주부, 온라인 쇼핑 경험 적음" https://shop.example.com
```

## 성격 랜덤 생성

매 실행마다 페르소나의 기본 정의에 아래 성격 축을 **랜덤 배정**하여 다양한 사용자를 시뮬레이션한다.

### Step 0.5: 성격 프로필 생성

실행 시작 시 아래 각 축에서 **하나씩 랜덤으로 선택**하여 이번 세션의 성격 프로필을 생성한다.

| 축 | 선택지 | 탐색 행동 영향 |
|----|--------|-------------|
| **IT 숙련도** | 초보, 보통, 능숙 | 초보: 메뉴/버튼을 잘 못 찾음. 능숙: 키보드 단축키나 URL 직접 입력 시도 |
| **탐색 성향** | 목적형, 탐험형 | 목적형: 최단 경로로 목표 달성. 탐험형: 여기저기 둘러보며 우연한 발견 |
| **인내심** | 낮음, 보통, 높음 | 낮음: 3초 이상 로딩이나 2번 이상 클릭 필요하면 포기. 높음: 끝까지 시도 |
| **나이대** | 10대, 20대, 30대, 40대+ | UI 기대치 다름. 40대+: 글자 크기, 대비에 민감 |
| **디바이스** | 데스크톱(1280x720), 모바일(390x844), 태블릿(768x1024) | viewport 변경 + 모바일 UX 검증 |
| **시간대** | 출근길(모바일, 급함), 점심시간, 퇴근 후(여유), 심야 | 인내심/탐색 성향에 영향 |

### 프로필 출력 예시

탐색 시작 전에 생성된 프로필을 출력한다:

```
================================
UX 탐색 세션 시작
================================
페르소나: 신규사용자
성격 프로필:
  - IT 숙련도: 초보
  - 탐색 성향: 탐험형
  - 인내심: 낮음
  - 나이대: 20대
  - 디바이스: 모바일 (390x844)
  - 시간대: 출근길 (급함)

목표: 이 서비스가 뭘 하는 곳인지 파악하고, 핵심 기능을 찾고 싶다

이 사용자는 IT에 익숙하지 않고, 여기저기 둘러보는 성향이지만
인내심이 낮아서 빨리 원하는 정보를 찾지 못하면 떠날 가능성이 높다.
모바일에서 출근길에 급하게 보고 있으므로 로딩 속도와 모바일 UI가 중요하다.
================================
```

### 성격이 행동에 미치는 영향

탐색 루프에서 성격 프로필을 참고하여 행동을 결정한다:

- **IT 초보 + 인내심 낮음**: 메뉴를 못 찾으면 즉시 "UX 이슈: 발견성 부족" 기록
- **IT 능숙 + 목적형**: URL을 직접 입력하거나 검색부터 시도
- **탐험형**: 홈에서 스크롤하며 여러 섹션을 둘러봄
- **모바일 디바이스**: viewport를 390x844로 변경, 햄버거 메뉴 사용
- **40대+ + 인내심 높음**: 글자 크기가 작아도 참고 읽지만, 이슈로 기록

## 워크플로우

### Step 0: 환경 준비

1. 시작 URL이 localhost면 dev 서버 실행 중인지 확인
2. 프로덕션 URL이면 바로 진행
3. 이 플러그인의 `scripts/` 디렉토리에 `node_modules`가 없으면 `npm install` 실행

### Step 0.5: 성격 프로필 생성

1. 위 성격 축에서 각각 하나씩 랜덤 선택
2. 프로필 요약 출력
3. 디바이스가 모바일이면 viewport 설정에 반영

### Step 1: Playwright 브라우저 실행

이 플러그인의 `scripts/explore.ts` 파일을 찾아서 실행한다. Glob 도구로 `**/roto-ux-explore/scripts/explore.ts` 패턴을 사용하여 경로를 찾는다.

```bash
npx tsx <explore.ts 경로> \
  --url="<시작URL>" \
  --screenshot-dir="/tmp/ux-explore-screenshots" \
  --headless=true
```

디바이스가 모바일이면 브라우저 실행 후 첫 명령으로 `viewport 390 844` 전송.

이 스크립트는:
- Chromium 브라우저를 실행
- 초기 페이지 로드 후 스크린샷 캡처
- 스크린샷 경로를 stdout으로 출력
- stdin으로 명령을 받아 실행 (click, type, scroll, navigate 등)
- 각 명령 실행 후 스크린샷 캡처 + stdout 출력

### Step 2: 탐색 루프 (최대 20단계)

각 단계에서:

1. **스크린샷 확인**: Read 도구로 현재 스크린샷 이미지를 확인
2. **상황 분석**: 페르소나 + 성격 프로필 관점에서 현재 화면을 분석
   - 이 성격의 사용자가 무엇을 볼까?
   - 어디를 클릭/입력하려 할까?
   - 혼란스러운 점은 없는가?
3. **UX 이슈 기록**: 발견한 문제를 기록 (성격 컨텍스트 포함)
   - "IT 초보 관점: 이 아이콘만으로는 기능을 유추하기 어려움"
   - "모바일에서 이 버튼이 너무 작아 터치하기 어려움"
   - "인내심 낮은 사용자: 여기서 원하는 정보를 찾지 못해 이탈할 수 있음"
4. **다음 행동 실행**: Bash로 explore.ts에 명령 전달
5. **새 스크린샷 확인** -> 2번으로 반복

인내심이 낮은 사용자는 목표를 달성하지 못하면 일찍 종료할 수 있다 (20단계 전에 "포기" 판정).

### Step 3: 리포트 생성

탐색 완료 후 아래 형식으로 리포트 출력:

```markdown
## UX 탐색 리포트

### 세션 정보
- **페르소나**: 신규사용자
- **성격**: IT 초보 / 탐험형 / 인내심 낮음 / 20대 / 모바일 / 출근길
- **목표**: 이 서비스가 뭘 하는 곳인지 파악하기
- **탐색 경로**: 홈 -> 스크롤 -> 메뉴 클릭 -> ...
- **총 단계**: 8 / 20 (인내심 부족으로 조기 종료)
- **대상 URL**: https://example.com

### 발견된 UX 이슈

#### 심각도: 높음
| # | 페이지 | 이슈 | 성격 컨텍스트 | 스크린샷 | 제안 |
|---|--------|------|-------------|---------|------|
| 1 | 홈 | 핵심 기능 진입점 불명확 | IT 초보는 네비게이션을 인식하지 못함 | step-3.png | 명확한 CTA 버튼 |

#### 심각도: 중간
...

#### 심각도: 낮음 / 제안
...

### 긍정적 발견
- ...

### 목표 달성 여부
- [x] / [ ] 단계별 체크
```

## explore.ts 브라우저 제어 스크립트

이 플러그인의 `scripts/explore.ts`는 Playwright 브라우저를 실행하고
stdin으로 명령을 받아 실행하는 REPL 서버이다.

### 지원 명령

| 명령 | 설명 | 예시 |
|------|------|------|
| `screenshot` | 현재 화면 캡처 | `screenshot` |
| `click <selector>` | 요소 클릭 | `click a[href="/about"]` |
| `click-text <text>` | 텍스트로 요소 찾아 클릭 | `click-text 시작하기` |
| `click-xy <x> <y>` | 좌표로 클릭 | `click-xy 200 300` |
| `type <selector> <text>` | 입력 필드에 텍스트 입력 | `type input[type=text] 검색어` |
| `scroll <direction>` | 페이지 스크롤 | `scroll down` |
| `navigate <url>` | URL 이동 | `navigate /about` |
| `wait <ms>` | 대기 | `wait 2000` |
| `viewport <width> <height>` | 뷰포트 변경 | `viewport 390 844` |
| `dom` | 현재 DOM 구조 요약 출력 | `dom` |
| `links` | 보이는 링크 목록 출력 | `links` |
| `inputs` | 보이는 입력/버튼 목록 출력 | `inputs` |
| `quit` | 브라우저 종료 | `quit` |

### 출력 형식

각 명령 실행 후:
```
[OK] click a[href="/about"]
[SCREENSHOT] /tmp/ux-explore-screenshots/step-003.png
[URL] http://localhost:3000/about
[TITLE] About | My App
```

## 반복 실행 모드 (--repeat=N)

`--repeat=N` 옵션을 주면 동일 페르소나로 N회 반복 탐색한다.
매 라운드마다 **새로운 랜덤 성격**이 배정되어 다양한 사용자 관점을 시뮬레이션한다.

### 반복 실행 흐름

```
Round 1: 성격 랜덤 생성 -> 탐색 -> 개별 보고서 작성
Round 2: 새 성격 랜덤 생성 -> 탐색 -> 개별 보고서 작성
...
Round N: 새 성격 랜덤 생성 -> 탐색 -> 개별 보고서 작성
-> 통합 요약 보고서 생성
```

### 보고서 구조

반복 실행 시 프로젝트 루트에 아래 파일들이 생성된다:

```
ux-explore-reports/
  {YYYY-MM-DD_HH-mm}/
    round-1.md             # Round 1 개별 보고서
    round-2.md             # Round 2 개별 보고서
    ...
    round-N.md             # Round N 개별 보고서
    summary.md             # 통합 요약 보고서
    screenshots/
      round-1/step-001.png
      round-2/step-001.png
      ...
```

### 통합 요약 보고서 (summary.md)

모든 라운드의 결과를 종합하여 아래 형식으로 작성:

```markdown
## UX 탐색 통합 보고서

### 세션 정보
- **페르소나**: 신규사용자
- **반복 횟수**: 5회
- **대상 URL**: https://example.com
- **실행 일시**: 2026-04-09 01:30

### 라운드별 성격 요약

| Round | IT 숙련도 | 탐색 성향 | 인내심 | 디바이스 | 목표 달성 | 이슈 수 |
|-------|----------|----------|--------|---------|----------|--------|
| 1 | 초보 | 탐험형 | 낮음 | 모바일 | 부분 달성 | 3 |
| 2 | 능숙 | 목적형 | 높음 | 데스크톱 | 달성 | 1 |
| ... | ... | ... | ... | ... | ... | ... |

### 이슈 빈도 분석

여러 라운드에서 반복 발견된 이슈를 빈도순으로 정리:

| 이슈 | 발생 빈도 | 관련 라운드 | 평균 심각도 | 제안 |
|------|----------|-----------|-----------|------|
| 네비게이션 진입점 불명확 | 3/5 | R1, R3, R5 | 중간 | CTA 버튼 추가 |

### 디바이스별 이슈 분류

| 이슈 | 데스크톱 | 모바일 | 태블릿 |
|------|---------|--------|--------|
| 네비게이션 | - | O | - |

### 성격별 인사이트

- **IT 초보 라운드에서만 발견된 이슈**: ...
- **인내심 낮음 라운드에서 조기 이탈한 지점**: ...
- **모바일에서만 발생한 이슈**: ...

### 우선순위 권장사항

빈도와 심각도를 기준으로 개선 우선순위:
1. [높음/빈도 4/5] ...
2. [중간/빈도 3/5] ...
```

### 단일 실행 보고서

`--repeat` 없이 실행하면 대화 내에 직접 리포트를 출력한다 (파일 미생성).
`--repeat=1`로 실행하면 파일로도 저장한다.

## 주의사항

- **최대 20단계**: 무한 루프 방지. 인내심 낮은 페르소나는 더 일찍 종료 가능
- **비파괴**: 로그인이 필요한 작업은 테스트 계정 사용 또는 읽기 전용으로 제한
- **프로덕션 안전**: 프로덕션 URL에서도 읽기 전용 탐색만 수행. 데이터 변경 금지
- **스크린샷 정리**: 탐색 완료 후 `/tmp/ux-explore-screenshots/` 정리

ARGUMENTS: $ARGUMENTS
